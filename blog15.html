JavaScript Arrays Simplified

If you‚Äôre new to JavaScript, you‚Äôll get to a point where you‚Äôll have to understand the concepts behind arrays, it‚Äôs implementations and how it works; which is why I‚Äôve taken my time to break everything down in this article to explain in-depth everything you need know and understand about arrays in JavaScript programming language.

Arrays just like any other programming language, enables storing a collection of multiple items under a single variable name and has members for performing common array operations.

Some Characteristics of JavaScript Arrays

  1. They are resizable and can contain a mix of different data types

  2. They are not associative arrays and so, array elements cannot be accessed using arbitrary strings as indexes, but must be accessed using non-negative integers (or their respective string form) as indexes.

  3. They are zero-indexed which means that the first element of an array is at index 0 while the last element is at the value of the array‚Äôs length property minus 1.

  4. JavaScript array-copy operations create shallow copies.

Note: All standard built-in copy operations with any JavaScript objects create shallow copies, rather than deep copies.

-- ARRAY INDICES

JavaScript syntax requires properties beginning with a digit to be accessed using bracket [] notation instead of dot notation. You can as well quote the array indices for instance years[‚Äò5‚Äô] instead of years[5], though they are usually not necessary.

But years[5] and years[05] would refer to two different slots on the years object.

                            
                              console.log(years["5"] != years["05"]); // indices 2 is not equal to 02

Only years[‚Äú5‚Äù] is an actual array index. years[‚Äú05‚Äù] is an arbitrary string property that will not be visited in array iteration.

Relationship between length and numerical properties

JavaScript array‚Äôs length property and numerical properties are connected.

Array Methods and Empty slots

  • Methods that do an in check before accessing the index and do not conflate empty slots with undefined:

    concat(); copyWithin(); every(); filter(); flat(); flatMap(); forEach(); indexOf(); lastIndexOf(); map(); reduce(); reduceRight(); reverse(); slice(); some(); sort(); splice();

  • Methods that treat empty slots as if they are undefined:

    entries(); fill(); find(); findIndex(); findLast(); findLastIndex(); group(); groupToMap(); includes(); join(); keys(); toLocalString(); values();

  • Copying methods and mutating methods:

    concat(), filter(), flat(), flatMap(), map(), slice(), splice()

  • Creating new arrays with the Array base constructor:

    toReversed(), toSorted(), toSpliced(), with()

CONSTRUCTOR

Array() ‚Äî Creates a new Array object.


STATIC PROPERTIES

Array[@@special] ‚Äî Returns the Array constructor


STATIC METHODS

  • Array.from() ‚Äî Creates a new Array instance from an iterable or array-like object

  • Array.fromAsync() ‚Äî Returns a new Array instance from an async iterable, iterable, or array-like object.

  • Array.isArray() ‚Äî Returns true if the argument is an array, or false otherwise.

  • Array.of() ‚Äî Creates a new Array instance with a variable number of arguments, regardless of number or type of the arguments.

INSTANCE PROPERTIES

These properties are defined on Array.prototype and shared by all Array instances.

  • Array.prototype.constructor ‚Äî The constructor function that created the instance object. For Array instances, the initial value is the Array constructor.

  • Array.prototype[@@unscopables] ‚Äî Contains property names that were not included in the ECMAScript standard prior to the ES2015 version and that are ignored for with statement-binding purposes.

  • Length ‚Äî Reflects the number of elements in an array.

INSTANCE METHODS

  1. Array.prototype.at() ‚Äî Returns the array item at the given index. Accepts negative integers, which count back from the last item.

  2. Array.prototype.concat() ‚Äî Returns a new array that is the calling array joined with other array(s) and/or value(s).

  3. Array.prototype.copyWithin() ‚Äî Copies a sequence of array elements within an array.

  4. Array.prototype.entries() ‚Äî Returns a new array iterator object that contains the key/value pairs for each index in an array.

  5. Array.prototype.every() ‚Äî Returns true if every element in the calling array satisfies the testing function.

  6. Array.prototype.fill() ‚Äî Fills all the elements of an array from a start index to an end index with a static value.

  7. Array.prototype.filter() ‚Äî Returns a new array containing all elements of the calling array for which the provided filtering function returns true.

  8. Array.prototype.find() ‚Äî Returns the value of the first element in the array that satisfies the provided testing function, or undefined if not appropriate element is found.

  9. Array.prototype.findIndex() ‚Äî Returns the index of the first element in the array that satisfies the provided testing function, or -1 if no appropriate element was found.

  10. Array.prototype.findLast() ‚Äî Returns the value of the last element in the array that satisfies the provided testing function, or undefined if no appropriate element is found.

  11. Array.prototype.findLastIndex() ‚Äî Returns the index of the last element in the array that satisfies the provided testing function, or -1 if no appropriate element was found.

  12. Array.prototype.flat() ‚Äî Returns a new array with all sub-array elements concatenated into it recursively up to the specified depth.

  13. Array.prototype.flatMap() ‚Äî Returns a new array formed by applying a given callback function to each element of the calling array, and then flattening the result by one level.

  14. Array.prototype.forEach() ‚Äî Calls a function for each element in the calling array.

  15. Array.prototype.group() ‚Äî Groups the elements of an array into an object according to the strings returned by a test function.

  16. Array.prototype.groupToMap() ‚Äî Groups the elements of an array into a Map according to values returned by a test function.

  17. Array.prototype.includes() ‚Äî Determines whether the calling array contains a value, returning true or false as appropriate.

  18. Array.prototype.indexOf() ‚Äî Returns the first (least) index at which a given element can be found in the calling array.

  19. Array.prototype.join() ‚Äî Joins all elements of an array into a string.

  20. Array.prototype.keys() ‚Äî Returns a new array iterator that contains the keys for each index in the calling array.

  21. Array.prototype.lastIndexOf() ‚Äî Returns the last (greatest) index at which a given element can be found in the calling array, or -1 if none is found.

  22. Array.prototype.map() ‚Äî Returns a new array containing the results of invoking a function on every element in the calling array.

  23. Array.prototype.pop() ‚Äî Removes the last element from an array and returns that element.

  24. Array.prototype.push() ‚Äî Adds one or more elements to the end of an array, and returns the new length of the array.

  25. Array.prototype.reduce() ‚Äî Executes a user-supplied ‚Äúreducer‚Äù callback function on each element of the array (from left to right), to reduce it to a single value.

  26. Array.prototype.reduceRight() ‚Äî Executes a user-supplied ‚Äúreducer‚Äù callback function on each element of the array (from right to left), to reduce it to a single value.

  27. Array.prototype.reverse() ‚Äî Reverses the order of the elements of an array in place. (First becomes the last, last becomes first).

  28. Array.prototype.shift() ‚Äî Removes the first element from an array and returns that element.

  29. Array.prototype.slice() ‚Äî Extracts a section of the calling array and returns a new array.

  30. Array.prototype.some() ‚Äî Returns true if at least one element in the calling array satisfies the provided testing function.

  31. Array.prototype.sort() ‚Äî Sorts the elements of an array in place and returns the array.

  32. Array.prototype.splice() ‚Äî Adds and/or removes elements from an array.

  33. Array.prototype.toLocaleString() ‚Äî Returns a localized string representing the calling array and it‚Äôs elements. Overrides the Object.prototype.toLocaleString() method.

  34. Array.prototype.toReversed() ‚Äî Returns a new array with the elements in reversed order, without modifying the original array.

  35. Array.prototype.toSorted() ‚Äî Returns a new array with the elements sorted in ascending order, without modifying the original array.

  36. Array.prototype.toSpliced() ‚Äî Returns a new array with some elements removed and/or replaced at a given index, without modifying the original array.

  37. Array.prototupe.toString() ‚Äî Returns a string representing the calling array and its elements. Overrides the Object.prototype.toString() methods.

  38. Array.prototype.unshift() ‚Äî Adds one or more elements to the front of an array, and returns the new length of the array.

  39. Array.prototype.values() ‚Äî Returns a new array iterator object that contains the values for each index in the array.

  40. Array.prototype.with() ‚Äî Returns a new array with the element at the given index replaced with the given value, without modifying the original array.

  41. Array.prototype[@@iterator]() ‚Äî An alias for the values() method by default

Note: All built-in array-copy operations(spread, .from(), .slice() and .concat() create shallow copies). What this means is that if you‚Äôre assigning an existing array to a new variable, the new variable doesn‚Äôt create a copy of either the array or its elements. Instead, the new variable is just a reference or alias, to the original array; that is, the original array‚Äôs name and the new variable name are just two names for the exact same object which will always evaluate as strictly equivalent.

This means that anytime you make any changes to either the value of the original array or the value of the new variable, the other will change too and vice-versa.

In case you‚Äôre interested in learning the difference between shallow and deep copies in JavaScript, you can read more about it here.
