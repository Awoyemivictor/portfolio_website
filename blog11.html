Different Ways to Declare Functions in JavaScript

  • 12 April 2023
  • By Awoyemi Victor
  • 24 Comments

In this article, you‚Äôll learn the different ways you can declare functions in JavaScript. One of the coolest thing about JavaScript is that functions can be:

  • Passed into other functions

  • Stored in variables

  • Moved around like any other piece of data in JavaScript.

There are 2 ways to declare a function in JavaScript:

  • A. Known Declarations

  • B. Unknown (Anonymous) Declarations

-- FUNCTION DECLARATIONS

These are function declarations that has a name. This is the normal way of declaring a function in JavaScript. For instance:

                            
                              function myCalculator(a, b) {
                                total = a + b;
                                return total;
                              }
Some key takeaways:

  1. Function declarations allows for hoisting (which means that JavaScript will take all function keyword and hoist them up, up and up and says ‚Äúyou‚Äôre a function, you belong at the top of the file‚Äù.

    That means that anywhere you call the function, it‚Äôll be available to you.

  2. It allows only for explicit return (having to type the return keyword)

  3. There are curly brackets {}

  4. There‚Äôs a need for parenthesis () even with a single function parameter

-- UNKNOWN/ANONYMOUS FUNCTION DECLARATIONS

Anonymous functions are functions without a name. For instance:

                            
                              function(name) {
                                return `Your name is ${name}`;
                              }
Some key takeaways:

  1. It displays error if you try running it directly on the console

  2. Anonymous functions are only valid in some use cases like callbacks, IIFE (immediately invoked function expression)

Examples of Anonymous function Declarations:

METHOD 1 ‚Äî Variable Function Expressions

Function expressions are also called Variable expressions. It‚Äôs a method of storing a function inside a variable.

                            
                              const myCalculator = function(a, b) { 

                                total = a + b; 

                                return total; 

                              }
Some key takeaways:

  1. JavaScript does not hoist variable functions

  2. Variables functions are anonymous functions (which means they are functions without a name

  3. They are Explicitly returned (which means you have to type the return keyword)

  4. They make use of curly brackets for statements {}

  5. There is always a need for parenthesis () even with a single function parameter.


METHOD 2 ‚Äî Arrow Function Expressions

These are function expressions that make use of the fat arrow => for the return statements.

                            
                              const myCalculator = (a, b) => a + b;
Some key takeaways:

  1. Arrow function is also an anonymous function

  2. They are Implicitly returned (Not having to type the return keyword)

  3. They don‚Äôt make use of curly brackets for statements

  4. There‚Äôs no need of parenthesis () around a single function parameter


METHOD 3 ‚Äî IFFY Function Expressions (immediately invoked function expression)

                              
                                (function(a, b) { 

                                  total = a + b; 

                                  return total; 

                                })();
Some key takeaways:

  1. Function will be wrapped by parenthesis (parenthesis always run first in JavaScript)

  2. It returns the value which you can immediately run by putting parenthesis on the end

  3. It can as well takes in parameters


METHOD 4 ‚Äî Method Function Expressions

Method is simply a function that lives inside of an object.

                            
                              const calNum = { 

                                name: "victor", 

                                myAdd: function(a, b) { 

                                  total = a + b; 

                                  console.log(`${this.name.toUpperCase()}, Your result is ${total}`); 

                                }, 

                                // Short hand Method 

                                mySub(a, b) { 

                                  total = a - b; 

                                  console.log(`${this.victor.toUpperCase()}, Your result is ${total}`); 

                                }, 

                                // Arrow function 

                                myDiv: (a, b) => { 

                                  total = Math.floor(a / b); 

                                  // this.name won't work here because they take the parent scope of *this* 

                                  console.log(`Your result is ${total}`); 

                                }, 

                              };

This is a function that is passed as an argument to another function.
